# Neocities

Please don't look too closely at the commit history.

## Requirements

- [Hugo](https://gohugo.io) (extended) — `brew install hugo`
- [Go](https://go.dev) 1.22+ — `brew install go`

## Layout

```
archetypes/                 # frontmatter scaffolds for `hugo new`
cmd/render-poems/           # JSON poems -> static/poems/*.svg + manifest
content/
  feed/<slug>/index.md      # page bundles: post + its assets in one folder
  journal/<slug>/index.md
  books/<slug>/index.md
  movies/<slug>/index.md
data/
  barter.yaml               # barter items (offering + looking-for)
  poems/<slug>.json         # visitor-submitted poems (source of truth)
  poems_manifest.yaml       # generated by render-poems
layouts/                    # hugo templates
  _default/baseof.html      # shared shell (eye, tooltips, popovers)
  _default/list.html        # generic grid (books, movies)
  feed/list.html
  journal/list.html
  barter/list.html
  about/list.html
  guestbook/list.html
  index.html                # home (refrigerator + poems)
  partials/                 # shared snippets (card, popover, stars)
  shortcodes/video.html     # bundle-aware <video>
static/                     # passthrough: chrome assets + hand-coded pages
  assets/                   # styles.css, eyes, magnets
  welcome/, wnrs/, ...      # standalone hand-coded pages
public/                     # build output (gitignored)
```

## Creating posts

Each post is a page bundle — a folder containing `index.md` plus any
images/videos it references. Markdown can reference assets by bare
filename; Hugo resolves them relative to the bundle.

```markdown
---
title: My Post
date: 2026-04-22 13:15:00
thumbnail: some-image.jpg
---

body content here. ![](some-image.jpg)
```

- `feed` posts only need `date:`
- `books`/`movies` use `title`, `date`, `thumbnail`, `rating`, `author`
- video tags must use the shortcode so bundle paths resolve in popovers:
  `{{< video "clip.mp4" >}}`

## Barter

Items live in `data/barter.yaml`, not as page bundles. Each item has a
`side: offering | looking-for` field. Thumbnails go in `static/posts/`.

## Poems

Submissions are JSON files in `data/poems/<slug>.json`. The renderer
turns each one into `static/poems/<slug>.svg` and writes
`data/poems_manifest.yaml` (ordered, grouped by month) for the home
template to iterate.

## Usage

```
make build         # render poems + hugo build
make serve         # local dev server at http://localhost:1313
make clean         # rm public/ and resources/

make new-feed slug=foo
make new-journal slug=foo
make new-books slug=foo
make new-movies slug=foo
```

## Deployment

`.github/workflows/deploy.yml` builds and pushes to Neocities on every
push to `main`. Requires repo secret `NEOCITIES_TOKEN` (your API key from
neocities.org account settings).

Build artifacts (`public/`, `resources/`) are not committed — CI rebuilds
from source on every push.
