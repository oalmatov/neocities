# Hugo Port Implementation Plan

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax for tracking. Verification is build-output-driven, not TDD (this is a static site with no test framework). The "test" for each phase is "does `hugo` build cleanly and does the output match the previous `public/`?"

**Goal:** Replace the custom Python build (`build.py` + `templates.py`) with Hugo, restructure content to use page bundles and a data file for barter, and set up CI to deploy to Neocities on push to main.

**Architecture:**
- **Hugo** as the static site generator (single binary, Go templates)
- **Page bundles** (`content/<section>/<slug>/index.md` + assets next to them) for `feed`, `journal`, `books`, `movies`
- **Data file** (`data/barter.yaml`) for barter — no per-item markdown
- **Static passthrough** (`static/`) for hand-coded pages (`welcome`, `wnrs`, `v-day-w-vi`, `epiccoolgirl`, `sitemap`), site chrome assets, and pre-rendered poem SVGs
- **Pre-rendered SVGs** for poems: a `cmd/render-poems/main.go` reads `data/poems/*.json` and writes `static/poems/*.svg` + `data/poems_manifest.yaml`. CI runs this before `hugo`.
- **GitHub Actions** runs `go run ./cmd/render-poems && hugo && deploy-to-neocities` on push. `public/` and `resources/` are gitignored.

**Tech Stack:** Hugo, Go (for `cmd/render-poems`), GitHub Actions, Neocities API (via `bcomnes/deploy-to-neocities`).

## Phase 7: CI

### Task 7.1: Workflow file
**Files:**
- Create: `.github/workflows/deploy.yml`

```yaml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: '1.22' }
      - run: go run ./cmd/render-poems
      - uses: peaceiris/actions-hugo@v3
        with: { hugo-version: '0.130.0' }
      - run: hugo --minify
      - uses: bcomnes/deploy-to-neocities@v3
        with:
          api_token: ${{ secrets.NEOCITIES_TOKEN }}
          cleanup: true
          dist_dir: public
```

- [ ] Write workflow.
- [ ] User: add `NEOCITIES_TOKEN` to repo secrets (manual step).
- [ ] Commit.
