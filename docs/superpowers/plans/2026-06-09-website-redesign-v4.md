# Website Redesign v4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restyle the entire Hugo site to a black / cream / JetBrains-Mono terminal aesthetic with a 4-column dashboard homepage (that replaces the nav bar), global eye-gif + dithered-tile overlays, and a perpetual "Big Other" ticker.

**Architecture:** Hugo static site. Global chrome (fonts, tile layer, eye overlay, ticker) lives in `layouts/_default/baseof.html`; all visual styling lives in the single fingerprinted stylesheet `assets/styles.css`. The homepage (`layouts/index.html`) is rewritten to use `baseof` via `{{ define "main" }}` and renders 4 columns + a paginated feed. Poem SVGs are regenerated in the new style by `cmd/render-poems/main.go`.

**Tech Stack:** Hugo v0.161 (extended), Go (for `cmd/render-poems`), vanilla CSS/JS, Google Fonts (JetBrains Mono).

**Source of truth for markup/CSS:** the agreed mockups on disk:
- `.superpowers/brainstorm/51844-1781043323/content/pages.html` — index + books (generated by `build_pages.py`)
- `.superpowers/brainstorm/51844-1781043323/content/extras.html` — poems + journal + barter (generated by `build_poems.py`)

These mockups inline images as data-URIs and use static placeholder data; in the real site, images are `static/assets/*` paths and content comes from Hugo queries. Port the **CSS rules and DOM structure** from them; wire the **data** per the Hugo snippets below.

**Design tokens (CSS variables to define once in `:root`):**
```css
:root{
  --bg:#000; --text:#e9e4d6; --head:#f2ecdd; --muted:#8c8470; --accent:#7fa86b;
  --hairline:rgba(233,228,214,.16); --border:rgba(233,228,214,.4);
}
```

**Verification (every task):** `make build` must succeed (it runs `go run ./cmd/render-poems` then `hugo --minify --gc --cleanDestinationDir`). For visual checks run `make serve` and open the page. There is no unit-test harness; verification is a successful build + visual confirmation against the mockup.

func renderSVG(p Poem, _ string) string {
	var b strings.Builder
	fmt.Fprintf(&b,
		`<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 %d %d" preserveAspectRatio="xMidYMid meet">`,
		canvasWidth, canvasHeight)

}
